# AI 架构重构与优化任务

## 🎯 角色设定
你是一名拥有 10 年以上开发经验的高级软件架构师和 AI 工程师，精通服务端与客户端的混合架构设计、LangChain 生态体系，以及复杂业务领域的代码重构。

## 📋 任务目标
请对当前项目中的所有 AI 相关业务代码进行系统性重构与进阶优化。
核心目标是：**将分散的 AI 功能统一使用 LangChain 框架进行重构，并下沉封装到独立且高可复用的 `packages/ai` 库中**。

## 💡 核心优化要求

1. **全面引入 LangChain 框架**：
   - 使用 LangChain（如 `@langchain/core` 等相关生态）重构现有的 AI 功能调用逻辑。
   - 利用 LangChain 的 Chain, Agent, Memory, PromptTemplate/ChatPromptTemplate 等核心组件来标准化 AI 业务流。

2. **核心模块统一抽象至 `packages/ai`**：
   - 所有的 AI 对话底层功能、大模型客户端（LLM Clients）、提示词管理、以及 LangChain 的核心处理流，**必须**集成到 `packages/ai` 库下。
   - **为服务端复用而设计**：`packages/ai` 内部的代码必须保持纯净（例如：绝对不能直接耦合特定的浏览器扩展 API 或业务端特定的全局状态），确保未来可以无缝在 Node.js 等纯服务端环境中被直接引用。

3. **严格遵守重构规范**：
   - 必须严格遵循文档 `docs/ai-work/refactor2.md` 中定义的架构与代码设计规范。
   - 识别当前代码的 Code Smells，坚决消除高耦合代码、长函数、大量 `if-else` 以及重复代码。
   - 运用合适的设计模式（如 Factory Method、Strategy 策略模式、Facade 外观模式等）来管理不同的 LLM 提供商（Provider）或多样化的交互情景。
   - 确保代码满足 SOLID 原则，高内聚低耦合，具备极强的可扩展性、可读性和可测试性。

## 🛠️ 执行步骤

请仔细分析现有代码库并按照以下步骤开展工作：

1. **代码现状与架构瓶颈分析**：扫描当前项目中所有与 AI 交互相关的代码，指出当前的架构缺陷（Code Smells）以及与重构目标的差异。
2. **LangChain 融合方案设计**：规划如何将现有流程无损映射到 LangChain 的相关概念（LLMs/ChatModels, Prompts, Output Parsers, Runnables/Chains）。
3. **`packages/ai` 模块架构规划**：规划 `packages/ai` 的内部目录结构、模块边界划分以及对外防腐暴露的 API（API Facade 设计）。
4. **实施重构与迁移**：提供重构落地代码，确保对外业务逻辑的黑盒一致性。

## 📤 期望输出格式

请按照以下结构组织你的汇报和重构代码方案：

1️⃣ **现状分析与 Code Smells**：列出当前 AI 相关代码的痛点以及违反的架构原则。
2️⃣ **模块架构与包结构设计**：说明 `packages/ai` 的目录层级结构，并论证其在未来直接跨到服务端复用的可行性。
3️⃣ **核心设计模式与 LangChain 映射**：深入分析选用了哪些设计模式（参照 `refactor2.md`），以及原有逻辑是如何封装为 LangChain 组件的。
4️⃣ **核心接口与类结构（UML 描述）**：针对重构后的 AI 模块，给出文字版类结构设计或 UML 图表。
5️⃣ **重构后的完整与关键代码**：分模块提供重构后 `packages/ai` 内部的核心封装代码，以及上层应用（业务侧）的调用代码。
6️⃣ **重构前后对比与总结**：提炼本次架构优化的核心价值点及未来演进建议。
